Chrome DevTools MCP 介绍

Chrome DevTools MCP 是由Google Chrome DevTools团队开发的专业浏览器自动化和调试工具，基于强大的Puppeteer框架构建，让AI编程助手能够直接控制Chrome浏览器并获取深度性能数据。

核心特性

性能分析专家

使用Chrome DevTools录制性能轨迹，提取可操作的性能洞察，包括Core Web Vitals分析

  • LCP、CLS、FID等关键指标监控
  • 网络时序瀑布图分析
  • JavaScript执行性能剖析

高级浏览器调试

分析网络请求、截图、检查浏览器控制台，提供专业级调试能力

  • 网络请求详细分析和过滤
  • 控制台消息捕获和检查
  • 页面截图和accessibility快照

可靠自动化

基于Puppeteer实现Chrome自动化，自动等待操作结果，确保稳定性

  • 智能等待机制，无需手动延迟
  • 异常处理和重试机制
  • 26个专业自动化工具
关于Puppeteer

Puppeteer 是Google官方开发的Node.js库，提供了控制Chrome浏览器的高级API。Chrome DevTools MCP正是基于这一成熟技术构建，具备以下核心能力：

  • 直接控制Chrome实例：启动、关闭、导航页面
  • 自动化用户交互：点击、输入、滚动等操作
  • 深度性能分析：利用Chrome DevTools Protocol获取详细性能数据
  • 网络监控：拦截和分析网络请求，提供完整的网络时序信息

这种基于Puppeteer的架构确保了与Chrome浏览器的深度集成和专业级的调试能力。

Chrome DevTools MCP 深度解析

高级配置

完整配置选项

{ "mcpServers": { "chrome-devtools": { "command": "npx", "args": [ "chrome-devtools-mcp@latest", "--channel=stable", // Chrome通道: stable/canary/beta/dev "--headless=false", // 是否无头模式 "--isolated=false", // 是否隔离模式 "--executablePath=/Applications/Google Chrome.app/Contents/MacOS/Google Chrome", "--browserUrl=http://localhost:9222", // 连接远程Chrome "--logFile=./chrome-devtools-debug.log" // 调试日志文件 ] } } }

参数详细说明

参数 说明 类型/可选值 默认值
--browserUrl, -u 连接到运行中的Chrome实例（使用端口转发） string 启动新实例
--headless 是否以无头模式运行（无UI界面） boolean false
--executablePath, -e 自定义Chrome可执行文件路径 string 系统默认路径
--isolated 创建临时用户数据目录，浏览器关闭后自动清理 boolean false
--channel 指定要使用的Chrome通道版本 stable, canary, beta, dev stable
--logFile 调试日志文件路径，设置DEBUG=*启用详细日志 string 不输出日志

用户数据目录

Chrome DevTools MCP使用持久化用户数据目录保持登录状态：

  • Linux/MacOS: $HOME/.cache/chrome-devtools-mcp/chrome-profile-$CHANNEL
  • Windows: %HOMEPATH%/.cache/chrome-devtools-mcp/chrome-profile-$CHANNEL

会话保持

Cookie、LocalStorage、Session在重启后保持

多实例共享

所有chrome-devtools-mcp实例共享同一数据目录

隔离模式

使用 --isolated=true 可创建临时目录，关闭后自动清理

决策建议

选择原则：

  1. 专业性要求高 → Chrome DevTools MCP（性能分析专家工具）
  2. 创新AI应用 → Chrome MCP（独特AI功能，适合探索）
  3. 企业级项目 → Playwright MCP（完整企业功能，长期支持）

综合考虑因素：项目规模、团队技术栈、维护成本、安全要求、长期规划
